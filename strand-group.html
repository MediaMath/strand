strand-group

A mechanism for grouping components - for button bars, search groups, and radio groups. Button bars and radio groups will respond with respond with 'changed' and 'selected' states and events. Currently, all 'mixed' component grouping mechanisms are for positioning and styling concerns only.

Public Properties

Property Description
selectedIndex : Number  
default : true
options : none
attribute : N/A
reflected to attribute : no 		An integer representing the index of the selected list item from the dropdown panel. As the list items will be filtered upon user interaction - the number returned will correspond with the index of the item in original data set (see 'data' for more information).
items : Array  (read-only)
default : []
options : none
attribute : N/A
reflected to attribute : no 		Returns a list of the items that may be selected. In some case these will be data objects, and light dom children in others.
fitparent : Boolen (optional) 
default : false
options : none
attribute : fitparent
reflected to attribute : yes 		Sets the width of the component to fill the width of it's parent element.
group : String  
default : horizontal
options : horizontalvertical
attribute : group
reflected to attribute : yes 		Sets the name of the group, which in turn sets the group attribute on it's light dom children. If no group name is specified, a group name will be
align : Boolean (optional) 
default : horizontal
options : horizontalvertical
attribute : align
reflected to attribute : yes 		Specifies the direction in which the group should be laid out (horizontal or vertical).
value : String  
default : none
options : none
attribute : value
reflected to attribute : no 		The value of the selected item within the group i.e.- the value of a selected strand-radio or strand-button.
