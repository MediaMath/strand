mm-ajax

A simple non rendered XMLHttpRequest wrapper api.

Example HTML Editor

<template is="auto-binding"> <mm-ajax method="GET" url="//mediamath.github.io/strand/index.html" response="{{resp}}" auto> </mm-ajax> <mm-scroll-panel style="width:100%; height: 120px"> <mm-header size="tiny">data:</mm-header> {{resp}} </mm-scroll-panel> </template>

Example Output

Attributes

Attribute Options
contentType : String (Optional) Default value: application/x-www-form-urlencoded Content type header to send to server none
method : String (Optional) Default value: GET http request method to use for XHR call. Note that the component exposes constants for dev usage. 'GET''POST''PUT''DELETE'
auto : Boolean (Optional) Default value: false when auto is true exec() is called whenever the body changes 'true''false'
debug : Boolean (Optional) Default value: false Show debugging information inline instead of hiding the component. 'true''false'
body : Object (Optional) The request body to pass to the server for POST or PUT calls none
response : Object (or any primitive) The response from the server none
withCredentials : Boolean (Optional) Default value: false Passes the withCredentials param to the XHR needed for some secure CORS scenarios. 'true''false'
timeout : Boolean (Optional) Default value: 10000 XHR timeout in milliseconds none
status : String (readonly) Readonly XHR object status string 'true''false'
state : Boolean (readonly) 'true''false'
xhr : Object (readonly) (Optional) A reference to the XHR object created for this call if needed none
promise : Object (readonly) returns an A+ compatible promise (pinkyswear) that may be used for request chaining none

Public Methods

Method Arguements
reset() creates a new XHR and promise object so that a 2nd call can be made. Also clears all settings. none
exec() Triggers the XHR call. none
abort() Cancels an in progres XHR call. none
addHeader(namevalue) Adds a header name/value pair to the headers array. name : String Default value: undefined The name of the header eg: X-Requested-By value : String Default value: undefined The value of the header
addParam(namevalue) Adds a queryString name/value pair to the params array. name : String Default value: undefined The name of the query param value : String Default value: undefined The value of the query param
addUrlParam(name) Adds a url chunk to the urlParams array. Useful for ids or other dynamic url parts. / is added automatically. name : String Default value: undefined The name of the url chunk to add.

Events

Event Event Detail
Type : 'data-abort'
Type : 'data-progress' percent : Number Decimal percentage of the progress. From 0 to 1. total : Number The total size in bytes of the request. current : Number The current number of bytes loaded.
Type : 'data-error' error : String A message about what has failed instance : Object A reference to the mm-ajax instance that failed response : Object (optional) In cases where the response has actually been recieved before the error this will contain the API response if any.
Type : 'data-result' This is fired when the server responds but before JSON or blob parsing has occured.
Type : 'data-ready' This is fired when the data is parsed and available on the response property response : Object The parsed response from the server instance : Object A reference to the mm-ajax component that requested the response
