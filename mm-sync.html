mm-sync

High level AJAX Component

Example HTML Editor

<template is="auto-binding"> <mm-sync id="search" endpoint="https://api.github.com/search/code?q=polymer+in:file+language:js+repo:polymer/polymer" auto> </mm-sync> <template repeat="{{$.search.data.items}}"> {{name}} </template> </template>

Attributes

Attribute Options
endpoint : String Default value: // The base url (any urlParams are appended to this) to use for http calls. none
auto : Boolean/String (Optional) Determines if the component should sync to server on data/param changes. Save will respond to only data changes. Load will respond to only param changes. True will respond to both. 'true''false''load''save'
autoDebounce : Number (Optional) Number in milliseconds to debounce changes to data or params. Only 1 call will occur no matter how many changes happen in this duration. none
autoIgnoreNodes : Array (Optional) Default value: [] An array of strings containing the names of any param nodes to ignore when their values change. none
syncMode : String (Optional) Default value: single A string which determines how to handle collection data. By default it attempts to serialize all data and make a single call, but batching is supported for APIs that requrire individual model calls. 'single''multiple'
contentType : String (Optional) Default value: multipart/form-data Content type header to send none
cacheBuster : String (Optional) Default value: null A string to specify the cache buster name to append to queryStrings none
inputParams : Object (Optional) An object containing queryParams urlParams and headers arrays. This should be used to modify any of these values prior to making a call. Typically this is managed by an adapter superclass. none
outputParams : Object (Optional) An object containing queryParams urlParams and headers arrays. Similar to inputParams but used for POST, PUT and DELETE calls instead of GET. none
saveResponse : Boolean (Optional) Default value: true Determines if the api response should be persisted back to the data property. This is useful for throwing away server responses to POST calls where you do not want to overwrite the view model none
csrf : Boolean (Optional) Default value: true When true, sync will attempt to locate X-CSRF-Token headers and store them for subsequent calls. none
csrfHeader : string The value of any captured CSRF headers none
cacheCsrf : Boolean (Optional) Default value: false If true, any captured CSRF headers will be cached to localStorage for later use across page reloads. 'true''false'
cacheGlobals : Boolean (Optional) Default value: false If true any globals set using setGlobals method will be cached to localStorage 'true''false'
data : Object (or primitive) (Optional) Default value: {} When using in an input capacity (get) this will be the server response. Alternately this is used to generate the body sent for POST/PUT/DELETE calls. none
silentData : Boolean (Optional) Default value: true Determines if updates to data property should trigger an event. 'true''false'
withCredentials : Boolean (Optional) Default value: false CORS withCredentials passthrough to mm-ajax 'true''false'
page : Number (Optional) Default value: 0 The current page for collections. This is used by adapters to provide paging support for APIs in a standardized way. none
pageSize : Number (Optional) Default value: 10 The number of records in a page to retrieve from the server. Typically only used by adapters. none

Public Methods

Method Arguements
get() Shortcut method for GET calls none
post() Shortcut method for POST calls none
put() Shortcut method for PUT calls none
delete() Shortcut method for DELETE calls none
sync(mode) Request a sync with the server mode : String Default value: get string to specify the operational mode you want to use for syncing. (typically GET/POST)
setData(datasilent) Equvelant to setting instance.data, however this provides the option to not announce the change via dispatched event data : Object/Primitive Default value: null The data to set silent : Boolean Default value: false When true a changed event will not be dispatched when data is set.
setGlobal(keyvalue) Sets a global value that can be retrieved on any other sync or adapter instance. Useful for persisting a subset of user settings. key : String Default value: null Key to use for global variable (retrievable on any mm-sync or adapter instance. value : Any Value to use for global variable. Objects are ok for cached data as it is stringified before storage when nessary.
getGlobal(key) Retreive a global value from the global storage context key : string returns the value of the global or null if not found.

Events

Event Event Detail
Type : 'sync-pending' Fired when ajax setup is complete
Type : 'sync-ready' data : Object Response data from server
Type : 'sync-error' error : String Error description text result : Object Server result if any was recieved before error occured.
